# 📊 Open Source Audit Project – Git

## 📌 Introduction
This project is an audit and exploration of **Git**, one of the most widely used open-source tools in software development. Git is a **distributed version control system** that helps developers track code changes, manage different versions of a project, and collaborate with others efficiently.

I chose Git for this audit because it plays a huge role in modern software development and is used by individuals, teams, and large organizations around the world.

---

## 🧠 About Git
**Git** was created by **Linus Torvalds** in **2005** to support the development of the Linux kernel. Since then, it has become the standard version control system for developers worldwide.

Git allows users to:
- Keep track of project history
- Work on features independently using branches
- Merge changes safely
- Collaborate with multiple contributors
- Restore previous versions when needed

---

The main goals of this project are to:

- Study an important open-source software tool

- Examine Git’s impact in the real world
- Practice basic **Linux shell scripting**

## 🚀 Key Features of Git
Git provides many powerful features that make software development easier and more organized:

### 1. Distributed Version Control
Every user has a complete copy of the repository, including its history. This makes Git reliable and flexible.

### 2. Fast and Efficient
Git is designed for speed. Most operations, such as commits, branching, and viewing history, are performed quickly.

### 3. Branching and Merging
Git makes it easy to create branches for new features or bug fixes and later merge them into the main project.

### 4. Data Integrity
Git protects project history using checksums, which helps ensure that files and commits are not accidentally corrupted.

### 5. Collaboration Support
Git allows many developers to work on the same project at the same time without overwriting each other’s work.

### What this means:
The GPL gives users the freedom to:
- Use the software for any purpose
- Study how the software works
- Modify the source code
- Share copies with others
- Distribute modified versions

This license supports the values of open-source software by making sure the code remains open and accessible.

## 💡 Open Source Philosophy
Git reflects the core ideas of the open-source movement:

### Transparency
Its source code is publicly available, so anyone can inspect how it works.

### Collaboration
Developers from around the world can contribute improvements, report bugs, and share ideas.

### Freedom to Innovate
Users are free to modify Git and adapt it to their own needs, which encourages creativity and growth.
